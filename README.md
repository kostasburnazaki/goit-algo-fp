# Фінальний проєкт: Алгоритми та структури даних

Цей проєкт містить комплексне практичне дослідження фундаментальних алгоритмів та структур даних, реалізоване на мові Python. Робота охоплює теми зв'язних списків, рекурсивних фракталів, обходу графів та дерев, черг із пріоритетами, динамічного програмування та імітаційного моделювання методом Монте-Карло.

---

## 📂 Структура проєкту та опис завдань

### Завдання 1: Однозв'язний список
Реалізовано базову структуру даних однозв'язного списку з наступними алгоритмами управління:
* **Реверсування (Reverse):** Функція, що змінює посилання між вузлами "на місці" (In-place за допомогою підходу Two Pointers) для повного розвертання списку без виділення додаткової пам'яті. Часова складність: $O(n)$.
* **Сортування (Merge Sort):** Ефективний алгоритм сортування злиттям, адаптований для роботи з покажчиками зв'язного списку. Використовує рекурсивний поділ списку навпіл за принципом "розділяй і володарюй". Часова складність: $O(n \log n)$.
* **Злиття (Merge lists):** Функція лінійного об'єднання двох попередньо відсортованих списків в один спільний відсортований список шляхом порівняння значень поточних вузлів. Часова складність: $O(n + m)$, де $n$ та $m$ — довжини списків.

### Завдання 2: Фрактал "Дерево Піфагора"
Реалізовано алгоритм побудови геометричного рекурсивного фракталу "Дерево Піфагора". Програма використовує бібліотеку `matplotlib` для обчислення та малювання лінійних площинних трансформацій під ухилом 45 градусів (прямокутний рівнобедрений трикутник над кожним базовим квадратом). Кожна гілка рекурсивно розгалужується на дві менші, утворюючи класичну фрактальну структуру. Користувач має можливість динамічно вказувати бажану глибину (рівень рекурсії) через консольне введення.

### Завдання 3: Алгоритм Дейкстри з бінарною купою
Програмна реалізація класичного пошуку найкоротших шляхів у зваженому орієнтованому/неорієнтованому графі від однієї стартової вершини до всіх інших. Для оптимізації вибору наступної вершини з мінімальною вагою (релаксація ребер) використано структуру даних **мінімальна бінарна купа (черга пріоритетів)** за допомогою вбудованого модуля `heapq`. Завдяки цій пірамідальній оптимізації загальна часова складність алгоритму становить $O((V + E) \log V)$, де $V$ — кількість вершин, а $E$ — кількість ребер графа.

### Завдання 4: Візуалізація бінарної купи
Програма будує деревоподібну ієрархію на основі лінійного масиву мінімальної купи (Min-Heap). За допомогою бібліотек `networkx` та `matplotlib` реалізовано графічне відображення зв'язків між батьківськими вузлами та їхніми нащадками. Для ідентифікації та зв'язування вузлів графа використовуються математичні індекси масиву купи ($2i + 1$ для лівого нащадка та $2i + 2$ для правого), що забезпечує точність структури без створення додаткових абстракцій (uuid).

### Завдання 5: Обхід дерева (DFS та BFS) з градієнтом
Реалізовано два класичні алгоритми обходу бінарного дерева:
* **DFS (Пошук у глибину):** Використовує структуру даних **стек**.
* **BFS (Пошук у ширину):** Використовує структуру даних **черга**.
* **Візуальна фішка:** Колір вузлів динамічно змінюється від темних (стартові точки) до світлих відтінків залежно від хронологічного порядку їхнього відвідування під час обходу.

### Завдання 6: Жадібний алгоритм vs Динамічне програмування
Порівняльний аналіз двох алгоритмічних підходів для розв'язання оптимізаційної задачі вибору страв (варіація задачі про рюкзак) за умов фіксованого бюджету:
* **Жадібний підхід:** Обирає страви на основі максимального коефіцієнту калорійності на одиницю вартості. Працює швидко ($O(n \log n)$), але не гарантує глобально оптимального розв'язку.
* **Динамічне програмування:** Прораховує оптимальну матрицю підзадач, гарантуючи 100% найкращий вибір для максимізації калорійності в межах заданого бюджету ($O(n \cdot \text{budget})$).
